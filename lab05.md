## Модуль 5: Агрегация и массовый сбор данных 

**Задание: настройте прокси Zabbix, создайте шаблон и примените его к хосту. Проверьте работу триггеров и шаблонов. Создайте вычисляемый элемент.** 

---
**План**  
- Настройте прокси Zabbix. 
- Создайте шаблон. 
- Примените шаблон к хосту. 
- Проверьте работу шаблонов. 

---
### Практическая работа

### 1. Настройка прокси Zabbix 
1. На сервере **zabbix-proxy**: установите репозиторий Zabbix, если до этого его еще не устанавливали.

2. Установите Заббикс прокси:
```bash
dnf install zabbix-proxy-pgsql zabbix-sql-scripts
```
3. Создайте базу данных:
```bash
sudo -u postgres createuser --pwprompt zabbix
sudo -u postgres createdb -O zabbix zabbix_proxy
```

4. На хосте **zabbix-proxy** импортируйте начальную схему и данные. Вам будет предложено ввести недавно созданный пароль. 
```bash
cat /usr/share/zabbix-sql-scripts/postgresql/proxy.sql | sudo -u zabbix psql zabbix_proxy 
```   
5. Настройте базу данных для Zabbix прокси. Отредактируйте файл /etc/zabbix/zabbix_proxy.conf
```ini
ProxyMode=0
Server=10.0.20.10
DBPassword=zabbix
Hostname=zabbix-proxy
```
6. Запустите процесс Zabbix прокси и настройте его запуск при загрузке ОС: 
```bash
systemctl restart zabbix-proxy
systemctl enable zabbix-proxy
```
7. **Добавление прокси**. 
Для настройки прокси в веб-интерфейсе Zabbix: 
- перейдите в: **Администрирование → Прокси**
- нажмите на **Создать прокси**
8. Заполните поля:
- Имя прокси: zabbix-proxy
- Режим прокси: Активный
- Адрес прокси: 10.0.10.50
И нажмите **Добавить**

9. На Zabbix-сервере в конфигурационном файле `/etc/zabbix/zabbix_server.conf` должна быть строка:
```ini
ProxyConfigFrequency=10
ProxyDataFrequency=1
```
После внесения изменений перезапустите сервер:
```bash
systemctl restart zabbix-server
```
10. Откройте два окна и в каждом соответственно запустите вывод журнала zabbix-server и zabbix-proxy
```bash
tail -f /var/log/zabbix/zabbix_server.log
tail -f /var/log/zabbix/zabbix_proxy.log
```
11. Убедитесь, что в журнале появились сообщения об успешном подключении. Если вы видите ошибку типа:
```ini
Unable to connect to [10.0.20.10]:10051 [cannot connect to address: [13] Permission denied]
```
Значит проблема в SELinux на **zabbix-proxy**
Доустановите пакет `zabbix-selinux-policy` на zabbix-server.

12. В силу особенностей настройки маршрутизации на стенде, возможно, также придется исправить адрес прокси в web-интерфейсе zabbix-сервера. (Запросы на zabbix-server приходят с адреса шлюза 10.0.20.1)

13. Перейдите в: **Мониторинг → Узлы сети → infra** затем **Настройка: Узел сети**
14. Переключите **Наблюдение через** → **Прокси**  и   ниже в поле выберите **zabbix-proxy**.
15. Нажмите **Обновить**.
16. Так же настройте наблюдение и для **ws**, **WINCLIENT**.
17.  На **infra** измените содержимое конфигурации **/etc/zabbix/zabbix_agent2.conf**
Исправьте строку, указывающую на сервер:
```ini
ServerActive=10.0.10.50
```
1.  Перезапустите агент на **infra**.
2.  Перенастройте самостоятельно агенты на **ws** и **WINCLIENT**.
3.  Убедитесь, что у всех ранее настроеных узлов статус **Активные проверки** зеленый.
---    

### 2. Создание шаблона 
1. Для создания шаблона выполните следующее:
- перейдите в **Сбор данных → Шаблоны**
- нажмите на **Создать шаблон**
- измените атрибуты шаблона:
    - Имя шаблона: Linux SSH
    - Группы шаблонов: Templates/Operating systems
    Во вкладке **Теги**:
    - Имя: Student
    - Значение: familia
    
    Во вкладке **Макросы**:
    -  Имя: `{$SSH_USER}`
    -  Значение: devops
Сохраните изменения, нажав **Добавить**.

2. В списке шаблонов найдите **Linux SSH** и нажмите на **Элементы данных**.
3. Создайте новый элемент как это показано в **Лабораторной 2** в разделе "**6. Настройте сбор данных с использованием SSH проверок**" 
Имя элементу дайте 	**Check kernel via SSH**, а в команду впишите `uname -r`
---
### 3. Применение шаблона к хосту 
1. Перейдите в **Сбор данных → Узлы сети**. 
2. Выберите хост **zabbix-db** и в "Шаблоны" начните писать **SSH**, чтобы добавить **Linux SSH**. 
3. Добавьте ранее созданный шаблон к хосту. Сохраните изменения. 
4. Перейдите в **Мониторинг → Последние данные** и для узла **zabbix-db** укажите имя данных **Check kernel via SSH**, чтобы убедиться, что новые данные собираются.
---
### Лабораторная работа

1. Добавьте созданный шаблон Linux SSH ко всем хостам с операционной системой Linux
2. Перепишите информацию о ядрах на хостах в отдельный файл для отчета тренеру.
