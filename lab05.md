
Шаг 5: Настройка прокси Zabbix 
На сервере Fedora Linux установите пакеты Zabbix: sudo dnf install zabbix-proxy-mysql zabbix-sql-scripts. 
Настройте базу данных для Zabbix Proxy: mysql -uroot -p -e "create database zabbix_proxy character set utf8 collate utf8_bin;". 
Импортируйте начальные данные: zcat /usr/share/doc/zabbix-proxy-mysql/schema.sql.gz | mysql -uzabbix -p zabbix_proxy. 
Настройте конфигурационный файл Zabbix Proxy: /etc/zabbix/zabbix_proxy.conf. 
Запустите и настройте службу прокси Zabbix: sudo systemctl start zabbix-proxy, sudo systemctl enable zabbix-proxy. 

Шаг 6: Создание шаблона 
Перейдите в раздел "Конфигурация" и выберите "Шаблоны". 
Нажмите кнопку "Создать шаблон". 
Задайте имя шаблона и добавьте необходимые элементы. 
Сохраните изменения. 

Шаг 7: Применение шаблона к хосту 
На сервере Fedora Linux установите агента Zabbix: sudo dnf install zabbix-agent. 
Настройте файл конфигурации агента Zabbix: /etc/zabbix/zabbix_agentd.conf. 
Запустите и настройте службу агента Zabbix: sudo systemctl start zabbix-agent, sudo systemctl enable zabbix-agent. 
Перейдите в раздел "Конфигурация" и выберите "Хосты". 
Выберите нужный хост и перейдите на вкладку "Шаблоны". 
Добавьте ранее созданный шаблон к хосту. 
Сохраните изменения. 


Настройте сбор данных HTTP 

Первым шагом является настройка сбора данных HTTP в Zabbix 7. Для этого выполните следующие действия: 
Откройте Zabbix и войдите в систему с правами администратора. 
Перейдите в раздел Configuration и выберите Hosts. 
Выберите нужный хост или создайте новый. 
В разделе Items добавьте новый элемент, выбрав тип HTTP agent. 
Укажите целевые URL-адреса, с которых необходимо собирать данные, и настройте параметры запроса. 
Настройте частоту сбора данных (например, каждые 5 минут). 
Сохраните изменения и активируйте элемент. 

Настройте синтетический веб-мониторинг 

Синтетический веб-мониторинг в Zabbix позволяет эмулировать действия пользователей и проверять работоспособность веб-приложений. Для настройки выполните следующие шаги: 
В интерфейсе Zabbix перейдите в раздел Configuration и выберите Web. 
Создайте новый веб-сценарий, указав имя и хост. 
Добавьте шаги сценария, указав URL-адреса и действия (например, открытие страницы, заполнение формы и отправка). 
Настройте частоту выполнения тестов и укажите точки мониторинга. 
Сохраните сценарий и запустите мониторинг. 

Настройте мониторинг файлов журналов 

Мониторинг файлов журналов помогает отслеживать ошибки и предупреждения. Вот как это сделать в Zabbix 7: 
Откройте Zabbix и перейдите в раздел Configuration, затем выберите Hosts. 
Выберите хост или создайте новый. 
В разделе Items добавьте новый элемент, выбрав тип Log file. 
Укажите путь к файлу журнала на сервере и настройте правила для поиска определенных сообщений (например, ошибок или предупреждений). 
Укажите частоту проверки файлов журналов. 
Сохраните настройки и активируйте элемент. 

Проверьте работу всех элементов 

После настройки всех мониторинговых систем необходимо убедиться, что они работают корректно: 
Перейдите в дашборд Zabbix. 
Проверьте, поступают ли данные по HTTP-запросам. 
Убедитесь, что синтетические тесты выполняются и результаты фиксируются. 
Проверьте файлы журналов на наличие новых записей. 

Изучите полученные данные 

Далее, проанализируйте собранные данные, чтобы выявить возможные проблемы: 
Просмотрите отчеты о HTTP-запросах и выявите любые отклонения или ошибки. 
Анализируйте результаты синтетических тестов на наличие сбоев или замедлений. 
Изучите файлы журналов на предмет ошибок или предупреждений, требующих внимания. 

Подготовьте отчет о проделанной работе 

На последнем этапе подготовьте отчет для представления результатов мониторинга: 
Соберите все данные в единый документ. 
Опишите настройки, использованные для каждого типа мониторинга. 
Представьте анализ данных и выявленные проблемы.
