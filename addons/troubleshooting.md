# **Типичные проблемы Zabbix и шаги их решения**  

Zabbix — мощная система мониторинга, но при настройке могут возникать различные проблемы. Рассмотрим самые распространенные ошибки и способы их устранения.

---

## **1. Zabbix Agent не передает данные на сервер**  
### **Ошибка:**  
На сервере Zabbix в интерфейсе видно, что агент не передает данные, статусы элементов данных — `Not supported` или `ZBX_NOTSUPPORTED`.  

### **Шаги решения:**  
1️⃣ **Проверить, запущен ли агент**  
   ```bash
   systemctl status zabbix-agent
   ```
   Если агент не работает, запустить:
   ```bash
   systemctl start zabbix-agent
   ```

2️⃣ **Проверить настройки конфигурации**  
   Открыть `/etc/zabbix/zabbix_agentd.conf` и убедиться, что указан верный сервер:  
   ```
   Server=192.168.1.100
   ServerActive=192.168.1.100
   ```

3️⃣ **Проверить доступность сервера Zabbix с агента**  
   ```bash
   telnet 192.168.1.100 10051
   ```
   Если соединение не устанавливается — проверить сетевые правила (firewalld, iptables, SELinux).

4️⃣ **Перезапустить агент**  
   ```bash
   systemctl restart zabbix-agent
   ```

✅ **Результат**: Агент снова передает данные.

## **2. Ошибка аутентификации агента: "Zabbix agent unreachable"**  
### **Ошибка:**  
На сервере Zabbix агент отображается как `Unreachable` и не отвечает.  

### **Шаги решения:**  
1️⃣ Проверить, правильно ли указан IP-адрес агента в настройках узла сети Zabbix.  
2️⃣ Проверить, не блокирует ли доступ `firewalld` или `iptables`:  
   ```bash
   firewall-cmd --list-all
   ```
   Добавить правило, если нужно:  
   ```bash
   firewall-cmd --add-port=10050/tcp --permanent
   firewall-cmd --reload
   ```

3️⃣ Проверить `SELinux`:  
   ```bash
   setenforce 0
   ```

✅ **Результат**: Агент становится доступным.

## **3. Zabbix Proxy не передает данные на сервер**  
### **Ошибка:**  
Лог Zabbix Proxy содержит ошибку:  
```
cannot send proxy data to server at "10.0.20.10": connection is not allowed
```

### **Шаги решения:**  
1️⃣ Проверить `zabbix_server.conf` на сервере:  
   ```
   AllowProxy=1
   ```
   В ROS7 нет такого параметра, прокси должен быть добавлен в узлы сети.  

2️⃣ Проверить соединение с сервером:  
   ```bash
   telnet 10.0.20.10 10051
   ```

3️⃣ Проверить конфигурацию прокси (`zabbix_proxy.conf`):  
   ```
   Server=10.0.20.10
   ```
   Перезапустить:  
   ```bash
   systemctl restart zabbix-proxy
   ```

✅ **Результат**: Прокси передает данные.

## **4. Ошибка при подключении к PostgreSQL: SASL auth failed**  
### **Ошибка:**  
```
failed SASL auth (ВАЖНО: пользователь "zbx_monitor" не прошёл проверку подлинности)
```

### **Шаги решения:**  
1️⃣ Проверить, можно ли подключиться вручную:  
   ```bash
   psql -h 127.0.0.1 -U zbx_monitor -d postgres
   ```

2️⃣ Проверить `pg_hba.conf`:  
   ```
   host all zbx_monitor 0.0.0.0/0 md5
   ```
   Перезапустить PostgreSQL:  
   ```bash
   systemctl restart postgresql
   ```

✅ **Результат**: Доступ к базе восстановлен.

## **5. Ошибка "Value Mapping already exists"**  
### **Ошибка:**  
При добавлении нового элемента появляется сообщение:  
```
Элемент данных с ключом "ssh.run" уже существует у шаблона "Linux SSH".
```

### **Шаги решения:**  
1️⃣ Проверить, нет ли такого элемента в уже подключенных шаблонах.  
2️⃣ Использовать уникальные ключи или удалить конфликтующий элемент.  

✅ **Результат**: Конфликт устранен.

## **6. SNMP-трап не отправляется с Mikrotik**  
### **Ошибка:**  
Mikrotik не отправляет SNMP-ловушки на внешний сервер.  

### **Шаги решения:**  
1️⃣ Включить SNMP-трап в Mikrotik:  
   ```bash
   /snmp set enabled=yes trap-community=public trap-target=192.168.1.100 trap-version=2
   ```

2️⃣ Проверить на сервере, приходит ли трафик:  
   ```bash
   tcpdump -i eth0 port 162
   ```

✅ **Результат**: SNMP-трап успешно отправляется.

## **7. Zabbix не отправляет уведомления в Telegram**  
### **Ошибка:**  
Оповещения не приходят в Telegram.  

### **Шаги решения:**  
1️⃣ Проверить, что Media Type настроен правильно:  
   ```bash
   curl -s -X POST "https://api.telegram.org/bot<TOKEN>/sendMessage" -d chat_id=<CHAT_ID> -d text="Test"
   ```
   Если сообщение не приходит — проверить `TOKEN` и `CHAT_ID`.

✅ **Результат**: Уведомления в Telegram работают.

## **8. Ошибка "Unsupported item key" при создании элемента данных**  
### **Ошибка:**  
При добавлении нового элемента данных появляется сообщение:  
```
Unsupported item key for this item type.
```

### **Шаги решения:**  
1️⃣ Проверить, установлен ли нужный плагин в Zabbix Agent 2.  
2️⃣ Убедиться, что используется правильный ключ. Например, для PostgreSQL:  
   ```
   pgsql.bgwriter
   ```

✅ **Результат**: Элемент успешно добавлен.

---

## **9. Zabbix Web Interface не загружается после обновления**  
### **Ошибка:**  
После обновления Zabbix веб-интерфейс недоступен.  

### **Шаги решения:**  
1️⃣ Проверить логи Apache/Nginx:  
   ```bash
   tail -f /var/log/nginx/access.log
   ```

2️⃣ Проверить конфигурацию PHP (`php.ini`):  
   ```
   memory_limit = 256M
   post_max_size = 16M
   upload_max_filesize = 16M
   ```

3️⃣ Перезапустить веб-сервер:  
   ```bash
   systemctl restart nginx
   ```

✅ **Результат**: Интерфейс снова работает.
